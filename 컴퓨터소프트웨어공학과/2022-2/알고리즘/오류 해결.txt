오류 구글링 방법
1. 오류 구문을 복사 후 번역한다.
2. 오류 구문을 복사 후 구글링한다.
3. 디버깅을 진행하며 오류가 생긴 코드를 찾아내고 해당 코드 줄을 복사하여 구글링한다.
4. 책에서 해당 오류 관련 목차를 찾아 공부한다.

Q.1	개발 환경 구축 시 VS2017에 빌드 기능 없음

C.1	VS2017 다운로드시에 필요한 워크로드를 포함한 후 다운로드 하지 않음

Q.1	다운로드 시 VS community 2017 버전을 다운로드 받아야하며
	워크로드에서 "c++를 사용한 데스크톱 개발"을 체크한 후 다운로드를 진행한다.

Q.2	fopen, fscanf 오류 발생{
'fopen': This function or variable may be unsafe. Consider using fopen_s instead. To disable deprecation, use _CRT_SECURE_NO_WARNINGS. See online help for details.
}

C.2	컴파일러의 SDL검사 권장으로 인해 경고 또는 오류가 발생함

A.2	#define _CRT_SECURE_NO_WARNINGS 해당 문자를 SDL 검사가 적용되는 헤더파일 (즉, #include <stdio.h>)
	위에 전처리기를 추가해준다.
	여기서 주의할 점은 해당 문자를 SDL 검사가 적용되는 헤더파일 아래에 전처리기를 추가하면
	경고와 에러메시지가 그대로 나온다는 것이다.

Q.3	콘솔창 자동으로 닫힘

C.3	프로젝트 생성 시 응용 프로그램으로 생성하지 않아 build 할 때 flag가 달라져 자동으로 콘솔 닫힘

A.3.1	소스 코드 마지막에 system("pause") 를 추가한다.

A.3.2	[ https://blog.bsk.im/2013/10/24/visual-studio-console-remained/ ]를 참조한다
